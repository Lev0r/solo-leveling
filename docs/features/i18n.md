# Feature: Internationalization (i18n)

The default UI language is **Ukrainian (`uk`)**. English (`en`) is supported as a fallback and for code review / development convenience. Every user-visible string in the app goes through the i18n bundle — never inline.

## Library

`i18next` + `react-i18next`. Reasons:

- Industry standard, well-typed, small.
- Plurals, interpolation, lazy-loaded namespaces out of the box.
- Pairs cleanly with Vite for code-splitting locales.

## Layout

```
src/i18n/
├── index.ts              — i18next setup, language detection, persistence
├── locales/
│   ├── uk/
│   │   ├── common.json   — shared (buttons, errors, units)
│   │   ├── today.json    — Today's Workout screen
│   │   ├── timer.json    — Interval timer screen
│   │   ├── config.json   — JSON import/export
│   │   ├── settings.json — User settings
│   │   └── admin.json    — Admin screens
│   └── en/
│       └── (mirrored)
```

Each `features/<name>/` folder corresponds to a namespace.

## Language Resolution

Order of precedence on app boot:

1. `/users/{uid}.language` — explicit user preference (set on first sign-in, editable in settings).
2. `navigator.language` if it starts with `uk` or `en`.
3. Hardcoded default: `uk`.

The chosen language is persisted to `localStorage['lang']` so the boot screen (before Firestore loads) renders in the right language.

## Usage Pattern

```tsx
import { useTranslation } from 'react-i18next';

export function StartTimerButton() {
  const { t } = useTranslation('timer');
  return <button>{t('start')}</button>;
}
```

```json
// src/i18n/locales/uk/timer.json
{
  "start": "Старт",
  "phase": {
    "work": "РОБОТА",
    "rest": "ВІДПОЧИНОК",
    "complete": "ГОТОВО",
    "paused": "ПАУЗА"
  },
  "round": "{{current}} / {{total}}"
}
```

## Conventions

- One namespace per feature folder; shared keys live in `common`.
- Keys are dot-paths in English (`phase.work`) — translations are values.
- Never concatenate strings in code (`"Round " + n`). Use interpolation (`{{current}}`).
- Numbers / dates / units: format via `Intl.NumberFormat` / `Intl.DateTimeFormat` with the active locale.
- Pluralization: use i18next's `_one`, `_few`, `_many`, `_other` suffixes (Ukrainian has all four).

```json
{
  "rounds_one": "{{count}} підхід",
  "rounds_few": "{{count}} підходи",
  "rounds_many": "{{count}} підходів",
  "rounds_other": "{{count}} підходи"
}
```

## What Stays Untranslated

- Exercise names and routine labels are **data**, not UI strings. They come from the routine document as the author wrote them (Ukrainian, in the default program). Translating user-authored content is out of scope.
- Logs and JSON exports are language-neutral.

## Settings

`/settings` screen exposes a language switcher. Changing it:

1. Writes `language` to `/users/{uid}`.
2. Updates `localStorage['lang']`.
3. Calls `i18n.changeLanguage(...)`.

## Testing

- One Playwright smoke test per locale, checking the Today's Workout screen renders the right strings.
- Vitest covers the format helpers (date, plurals).

## Out of Scope (v1)

- Right-to-left languages
- Translating exercise content
- Time-of-day-sensitive greetings (e.g. "Good morning")
- Crowdsourced translations
